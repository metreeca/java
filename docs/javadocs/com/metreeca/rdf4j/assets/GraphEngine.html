      • create

        public Future<Response> create(Request request)
        Creates an LDP resource.

        Handles creation requests on the linked data container identified by the request focus item, according to the following operating modes.

        If the request target is a collection:

        • the request is expected to include a member resource shape;
        • the request RDF body is expected to contain an RDF description of the resource to be created matched by the shape using the request item as subject;
        • the resource to be created is assigned a unique IRI based on the stem of the the request IRI and the value of the Slug request header, if one is found, or a random UUID, otherwise;
        • the request RDF body is rewritten to the assigned IRI and stored into the shared graph ;
        • the target container identified by the request item is connected to the newly created resource as required by the LDP container profile identified by the filtering constraints in the request shape;
        • the operation is completed with a status code;
        • the IRI of the newly created resource is advertised through the Location HTTP response header.

        Otherwise:

        • the request is reported with a Response.InternalServerError status code.

      • relate

        public Future<Response> relate(Request request)
        Retrieves an LDP resource.

        Handles retrieval requests on the linked data resource identified by the request focus item, according to the following operating modes.

        If the focus item is a collection:

        • the request is expected to include a combined container/member shape;
        • the response includes the derived shape actually used in the retrieval process;
        • the response RDF body includes the RDF description of the container as matched by the Guard.Target area of request shape, linked using the ldp:contains property to the RDF description of the resources matched by the filtering constrains of the Guard.Digest area of the request shape;
        • Guard.Digest area of the request shape doesn't contain any filtering constrains, member resources explicitely to linked to the target container using the ldp:contains property are included;
        • if the request contains a filtering query, only matching container members descriptions are included.
        • if the request contains a Prefer header requesting the ldp:preferMinimalContainer representation, member descriptions are omitted;
        • the operation is completed with a status code.

        Otherwise, if the shared graph actually contains a resource matching the request item IRI:

        • the request is expected to include a member resource shape;
        • the response includes the derived shape actually used in the retrieval process;
        • the response RDF body contains the RDF description of the request item, as matched by the request request shape;
        • the operation is completed with a status code.

        Otherwise:

        • the operation is reported as unsuccessful with a status code .

        Regardless of the operating mode, RDF data is retrieved from the shared graph.

      • update

        public Future<Response> update(Request request)
        Updates an LDP resource.

        Handles updating requests on the linked data resource identified by the request item, according to the following operating modes.

        If the request target is a collection:

        • the request is reported with a Response.InternalServerError status code.

        Otherwise, if the shared graph actually contains a resource matching the request item IRI:

        • the request is expected to include a member resource shape;
        • the request RDF body is expected to contain an RDF description of the resource to be updated matched by the shape;
        • the existing RDF description of the target resource matched by the request shape is replaced in the shared graph with the request RDF body;
        • the operation is completed with a status code.

        Otherwise:

        • the operation is reported with a status code.

        Handles deletion requests on the linked data resource identified by the request item, according to the following operating modes.
