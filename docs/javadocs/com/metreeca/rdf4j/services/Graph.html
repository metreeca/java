Class Graph

All Implemented Interfaces:
AutoCloseable
public final class Graph extends Object implements AutoCloseable
Graph store.

Manages task execution on an RDF repository.

Nested task executions on the same thread will share the same connection to the backing RDF repository through a ThreadLocal scope variable.

    • configure

      @SafeVarargs public static <M extends Message<M>, O extends org.eclipse.rdf4j.query.Operation> O configure (M message, O operation, BiConsumer<M,O>... customizers)
      Configures standard bindings for SPARQL operations.

      Configures the following pre-defined bindings for the target SPARQL operation:

      pre-defined bindings
      variable value
      ?time an xsd:dateTime literal representing the execution system time with millisecond precision
      ?this the focus item of the filtered message
      ?stem the namespace of the IRI bound to the this variable
      ?name the local name of the IRI bound to the this variable
      ?task the HTTP method of the original request
      ?base the base IRI of the original request
      ?item the focus item of the original request
      ?user the IRI identifying the user submitting the original request or RDF.NIL if no user is authenticated

      If the target message is a response, the following additional bindings are configured:

      response bindings
      variable value
      ?code the HTTP status code of the filtered response
      Type Parameters:
      M - the type f the message to be filtered
      O - the type of the SPARQL operation to be configured
      Parameters:
      message - the message to be filtered
      operation - the SPARQL operation executed by the filter
      customizers - optional custom configuration setters for the SPARQL operation
      Returns:
      the input operation with standard bindings and optional custom configurations applied
      Throws:
      NullPointerException - if any argument is null or if customizers contains null values

    • update

      public <V> V update (Function<org.eclipse.rdf4j.repository.RepositoryConnection,V> update)
      Executes an update inside a transaction on this graph store.
      • if a transaction is not already active on the underlying storage, begins one and commits it on successful update completion;
      • if the update throws an exception, rolls back the transaction and rethrows the exception;
      • in either case, no action is taken if the transaction was already terminated inside the update.
      Type Parameters:
      V - the type of the value returned by update
      Parameters:
      update - the update to be executed
      Returns:
      the value returned by update
      Throws:
      NullPointerException - if update is null
