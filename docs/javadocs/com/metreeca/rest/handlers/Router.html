    Request router.
    Path-Based Routing

    Delegates request processing to a handler selected on the basis of the request HTTP path, ignoring the leading segment possibly already matched by wrapping routers.

    Requests are forwarded to a registered handler if their path is matched by an associated pattern defined by a sequence of steps according to the following rules:

    • the empty step (/) matches only an empty step (/);
    • literal steps /<step> match path steps verbatim;
    • wildcard steps /{} match a single path step;
    • placeholder steps /{<key>} match a single path step, adding the matched <key>/<step> entry to request parameters; the matched <step> name is URL-decoded before use;
    • prefix steps /* match one or more trailing path steps.

    Registered path patterns are tested in order of definition.

    Method-Based Routing

    If the route index doesn't contain a matching handler, delegates request processing to a handler selected on the basis of the request HTTP method.

    OPTIONS and HEAD methods are delegated to user-overridable default handlers.
