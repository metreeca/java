    • lift

      public T lift (Message<?> message)
      Lift a message part into this message.

      Mainly intended to be used inside wrappers to lift the main message part in multipart requests for further downstream processing, as for instance in: 

      handler -> request -> request.body(multipart(1000, 10_000)).fold(

     parts -> Optional.ofNullable(parts.get("main"))

         .map(main -> {

           ... // process ancillary body parts

           return handler.handle(request.lift(main));

         })

         .orElseGet(() -> request.reply(new Failure()
             .status(BadRequest)
             .cause("missing main body part")
         )),

     request::reply

 )
      Parameters:
      message - the message part to be lifted into this message
      Returns:
      this message modified as follows:
      • source message headers are copied to this message overriding existing matching values;
      • source message body representations are copied to this message replacing all existing values.
      Throws:
      NullPointerException - if message is null

      public T header (String name, String value)
      Configures message header value.

      If name is prefixed with a tilde (~), header values are set only if the header is not already defined; if name is Set-Cookie or is prefixed with a plus sign (+), header values are appended to existing values; otherwise, header values overwrite existing values.

      Parameters:
      name - the name of the header whose value is to be configured
      value - the new value for name; empty values are ignored
      Returns:
      this message
      Throws:
      NullPointerException - if either name or value is null
      See Also:
